No weather station is complete without a display, because who wants to see their weather predictions on a serial console? In this section, we will add the [CY8CKIT-028-TFT Display](https://www.cypress.com/documentation/development-kitsboards/tft-display-shield-board-cy8ckit-028-tft) to the project and use the [Segger emWin library](https://www.segger.com/products/user-interface/emwin/) to display some text and graphics.

## Install display

Create a new ModusToolbox project like before and use the Modus library manager to add support for `display-tft-st7789v` (under "Board Utils") and `emwin` (under "PSoC6 Middleware"). Then update your project.

![ModusToolbox Add libraries](/assets/images/weather_forecast/add_display_libs.png)

After that, follow the instructions in the [TFT ST7789V Repository](https://github.com/cypresssemiconductorco/display-tft-st7789v) to display a "Hello World" message on the display.

## Convert PNG image to C array

Now that we know how to display text, let us also add some graphics to our display. Unfortunately, our plain C program has no concept of a file system, so we need to contain our image in the program directly. What we instead have to do is convert the file to a simple array of bytes. The array can then be compiled directly into our program. This is what we will be doing in this section.

I downloaded icons for Rain, Snow, Clouds and Sun - the categories that the model will be able to predict - from [FlatIcon](https://www.flaticon.com/). The display resolution is 320x240px, so our image does not need to be larger than that. Considering the fact that there will be other things like the currently measured values on the display, it can even be considerably smaller, saving us some valuable Flash memory. I downloaded the images PNG option with a resolution of 128x128px. First, we need to convert the PNG to a bitmap file. The display library emWin would also support PNG files, but at the cost of having to install an additional library that does dynamic decompression in the microcontroller's RAM. To make things easier, we will be using BMP instead. BMP has the advantage that it can be read directly from Flash and written to the display, without the intermediate decompression step.

To convert the image, simply open it in an image editing program of your choice (MS Paint will do) and save it again as a Bitmap. There are Online Converters that do the same thing, though some of them might remove the Alpha Channel, leading to the Image being displayed in a colored box (which happened to me). MS Paint seems to work just fine.

Next, we convert the Bitmap image to a C array that we can then compile directly into our source code. The Unix command line tool `xxd` has support for this (if you're on Windows, like me while working on this project, you can use [WSL2](https://docs.microsoft.com/de-de/windows/wsl/install-win10)):

```bash
xxd -i /mnt/c/<path to bmp file>/cloud.bmp > cloud.c
```

We only need to make a few small changes to the generated file. The file generated by `xxd` should look like this

```c
unsigned char cloud_bmp[] = {
 	0x42, 0x4d, 0x8a, 0x00, 0x04, 0x00, 0x00, 0x00, 0x00, 0x00, 0x8a, 0x00,
 	// ... many, many lines ...
	0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00
};
unsigned int cloud_bmp_len = 262282;
```

We need to add a header file and declare the array as `const` so that it doesn't accidentally end up in RAM. This is what the file should look like after our changes:

```c
#include "cloud.h"

const unsigned char cloud_bmp[] = {
 	0x42, 0x4d, 0x8a, 0x00, 0x04, 0x00, 0x00, 0x00, 0x00, 0x00, 0x8a, 0x00,
 	// ... many, many lines ...
	0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00
};
const unsigned int cloud_bmp_len = 262282;
```

I'm sure there is an automated way of adding the extra lines by piping the output of `xxd` trough `sed` or `awk`. Feel free to send me your bash one-liners!

We also need a matching header file, which we will name `cloud.h`. It only needs to contain the declarations of the values in the C-file and an [Include Guard](https://en.wikipedia.org/wiki/Include_guard).

```c
#ifndef CLOUD_BMP_H_
#define CLOUD_BMP_H_

extern const unsigned char cloud_bmp[];
extern const unsigned int cloud_bmp_len;

#endif  // CLOUD_BMP_H_
```

## Display image

Move your Bitmap header and source files into your project directory. In your `main()` method, include your header file using `#include "cloud.h"`. At the start of your main method, add the following line to display your image in the middle of the TFT display:

```c
GUI_BMP_Draw(cloud_bmp, 96, 56);
```

Make sure you call this function *once* during setup and no inside your endless for-loop. Otherwise it will constantly be redrawn to the screen. When you compile and run your program, you should now see your image on the screen.

Once you compile and flash the program, you should see your image on the display:

In this article, we have:

- Installed the required display libraries to use the CY8CKIT-028-TFT display with our project

- Drawn the image to the display
